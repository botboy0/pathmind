<objective>
Fix `PathmindRuntimeImpl.getBlock(double, double, double)` so it returns `null` (Lua `nil`) for genuinely unloaded chunks instead of falling through to a real `minecraft:void_air` block id.

Purpose: The current gate uses only `client.world.isChunkLoaded(chunkX, chunkZ)`, which on the CLIENT returns `true` even for far/unloaded chunks. The null-return guard never fires, so the method reads `getBlockState` and returns real `void_air` — behaviorally confirmed in-game (UAT) at player.x + 5000 (~312 chunks out). This breaks the documented contract that `getBlock` returns `nil` for unloaded chunks.

Output: `getBlock` mirrors the proven working reference at `common/src/main/java/com/pathmind/nodes/Node.java:5508-5514`, which uses `getChunk(...)` then `chunk == null || chunk.isEmpty()` as the authoritative unloaded signal (client-side `getChunk` returns an `EmptyChunk` placeholder whose `isEmpty()` is `true` for unloaded chunks).
</objective>
