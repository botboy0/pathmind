## Phase Goal

**As a** Pathmind user, **I want to** call `pathmind.moveTo(x,y,z)` from my script and have it block until I arrive, **so that** I can sequence movement and subsequent logic linearly in Lua while the game keeps running smoothly.

This is slice S3 from RESEARCH.md. After this plan, `pathmind.moveTo` is a real awaitable that drives Baritone/PathmindNavigator and blocks only the worker thread.

<objective>
Implement awaitable movement: `PathmindRuntimeImpl.moveTo(x,y,z)` wraps `PathmindNavigator.startGoto(BlockPos, "Lua moveTo", CompletableFuture<Void>)` and returns that future; the `pathmind.moveTo` Lua binding blocks the worker on `navFuture.get()` (tick thread keeps running and completes the future on arrival), pauses the compute-time timeout clock while blocked, and converts navigation failure into a clear Lua error. Satisfies BIND-02.

Purpose: The async-sync bridge's headline capability — the worker blocks on a real game action without freezing the game (success criterion 2).
Output: Real moveTo impl; real moveTo binding with timeout pause/resume; an addon-side test using a controllable fake runtime to prove block-until-complete, failure→Lua-error, and that the timeout clock does not advance during the block.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Lua worker thread (blocked on nav CF) ↔ game tick thread (completes nav CF) | The worker blocks; the tick thread must remain free to drive navigation and complete the future. A blocking mistake here freezes the game. |
| Script → navigation resource | A script triggers real player movement via Baritone/PathmindNavigator. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-09 | Denial of Service | moveTo blocking the game tick thread instead of only the worker | mitigate | Blocking is exclusively `navFuture.get()` on the worker (off-thread); PathmindNavigator completes the future from the tick loop. Verified by the test completing the future from a separate thread and the worker unblocking. The game thread only polls the outer NodeResult future per tick (existing Node.java contract). |
| T-02-10 | Denial of Service | Long navigation falsely tripping the runaway-script timeout and killing a valid path | mitigate | Compute-time clock pauses while blocked on the nav future (CONTEXT decision); the InterruptHandler does not fire during Java-side blocking.
