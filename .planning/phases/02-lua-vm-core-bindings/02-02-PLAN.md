## Phase Goal

**As a** Pathmind user, **I want to** read and write node-tree variables from my Lua script, **so that** my script and the visual nodes around it share state (a script computes a value, a downstream Condition/Action node reads it).

This is slice S2 from RESEARCH.md. After this plan, `pathmind.getVar`/`pathmind.setVar` are real and round-trip scalars through Pathmind's existing runtime-variable store.

<objective>
Implement the variable half of the runtime bridge: real `getVariable`/`setVariable` in `PathmindRuntimeImpl` (wrapping `ExecutionManager.getRuntimeVariableFromAnyActiveChain` / `setRuntimeVariableForAnyActiveChain`), and wire the `pathmind.getVar`/`pathmind.setVar` Lua bindings to call them with scalar marshaling (number/string/boolean only — tables/userdata raise a Lua error per CONTEXT decision). Satisfies BIND-01.

Purpose: Make scripts exchange data with the rest of the graph — the first user-visible capability beyond "it runs".
Output: Real getVariable/setVariable; real getVar/setVar bindings; a Pathmind-side variable marshaling unit test; an addon-side round-trip test.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Lua value → Pathmind variable store | Script-supplied values cross into the shared runtime-variable map that other nodes read; type confusion here would corrupt downstream node logic. |
| Lua worker thread → ExecutionManager variable accessors | getVariable/setVariable run on the worker thread; ExecutionManager's accessors are the established thread-safe entry points. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Tampering | setVar writing an unsupported/complex type into the shared variable store | mitigate | Bindings accept only number/string/boolean; tables/functions/nil raise a Lua error before reaching the store. setVariable also throws IllegalArgumentException as a defense-in-depth backstop. Verified by the table-value test case. |
| T-02-07 | Tampering | Wrong NodeType/key marshaling corrupts cross-node variable reads | mitigate | Use only the verified key conventions (PARAM_AMOUNT→"Amount", PARAM_BOOLEAN→Mode/Toggle/Variable, PARAM_MESSAGE→"Text"); unit-tested round-trip + cross-store read assertion. |
| T-02-08 | Information Disclosure | Variable name collisions reading another chain's data | accept | Reads go through getRuntimeVariableFromAnyActiveChain which intentionally falls back to global scope — this is the existing, intended Pathmind variable-sharing semantics, not a new exposure (single-user client-side mod). |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new package-manager installs in this plan (Cobalt added in plan 01). Carry-forward of T-02-SC; nothing new to gate. |
