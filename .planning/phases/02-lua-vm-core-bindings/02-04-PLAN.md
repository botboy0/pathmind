<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Lua worker thread → MinecraftClient (client thread) | Game-state reads must execute on the client thread; the worker must never touch MinecraftClient directly. |
| Game-paused / stalled tick queue → worker waiting on result | If the tick queue stalls (game paused, screen open), the worker must not hang forever. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-13 | Tampering / crash | Reading MinecraftClient state off the client thread | mitigate | All three accessors dispatch via MinecraftClient.getInstance().execute and hand the result back through a CompletableFuture; the worker never touches MC directly. Verified by source (grep) + in-game UAT. |
| T-02-14 | Denial of Service | Worker hanging on getPosition/etc when the tick queue is stalled (game paused) | mitigate | Bounded `result.get(30, TimeUnit.SECONDS)` with a safe-default return on timeout (Pitfall 3). Not an indefinite block. |
| T-02-15 | Information Disclosure | Inventory/position/block exposes the player's own data to the player's own script | accept | Single-user client-side mod; the script author and the player are the same person. No new exposure. |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | No new package installs in this plan; carry-forward of T-02-SC. The UAT step republishes the Pathmind API artifact to mavenLocal (local dev artifact, not a registry package). |
</threat_model>
