<objective>
Close GAP-1 (major): an open addon palette category that overflows the sidebar has no scrollbar and its bottom entries are unreachable. Two root causes, both pre-diagnosed in 01-REVIEW.md:
- WR-03: `Sidebar.getCategoryScrollMetrics()` returns null whenever `selectedCategory == null`, so when an addon category is open (`selectedAddonCategory != null` but `selectedCategory == null`) no scrollbar is produced and the thumb-drag paths are dead.
- WR-04: the addon branch of `calculateMaxScroll` adds a flat `CATEGORY_HEADER_HEIGHT` plus `addonDefs.size() * NODE_HEIGHT`, while the render pass sizes the header as `Math.max(CATEGORY_HEADER_HEIGHT, headerLines.size() * headerLineHeight)` and each row as `Math.max(NODE_HEIGHT, lines.size() * nodeLineHeight + PADDING)`. For wrapping names the rendered height exceeds `maxScroll`, so the last rows fall outside the scrollable range.

Purpose: addon categories scroll exactly like built-in categories, proving the addon palette (LUA-01) is fully usable with realistic entry counts.
Output: addon-aware scroll metrics, wrap-aware addon content height, an addon-pass scrollbar mirroring the built-in pass, and a unit test for the height computation.
</objective>
