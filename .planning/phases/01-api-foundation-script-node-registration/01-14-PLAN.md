<objective>
Close the 3 editor-render gaps found in in-game UAT round 5 (VERIFICATION.md `uat_round_5.gaps_found`). All three live on the addon-node editor-render surface and are wiring/branch defects in already-shipped code — no new subsystems.

- **UAT-GAP-B**: the palette drag-preview/ghost title renders `NodeType.ADDON.getDisplayName()` ("Addon Node") instead of the addon's registered display name. The placed-node title path was already fixed (Plan 01-08, `Node.getDisplayName()` ADDON branch), but the drag-preview path in `PathmindVisualEditorScreen` does not route through it.
- **UAT-GAP-C**: a registered addon node dragged over the sidebar hits the GAP-4 sidebar-overlap-suppression branch, which reuses `renderAddonPlaceholderBody` — and that method unconditionally draws "⚠ addon missing". A present, registered addon must NEVER show the missing-addon placeholder; it must use the standard dimmed/invalid-drop treatment.
- **UAT-GAP-A**: the "Scripting" addon category shows no scrollbar in-game. Root cause is twofold: `calculateMaxScroll` appends a magic `+100` that diverges from the tested `computeAddonContentHeight` helper (WR-03), and with a single Script entry the category does not overflow — so no scrollbar is the CORRECT behavior. This task confirms the no-overflow case (no scrollbar) and wires the overflow case (scrollbar shows) both through the same wrap-aware formula, then locks both with tests.

Purpose: LUA-01 (grab Script node from palette and place it) and API-07 (addon node body render hook) must look correct in-game while dragging and while hovering invalid drop targets — the current visuals lie about the addon being missing and mislabel the node.
Output: Corrected drag-preview title (3 compat copies), a neutral-body branch for resolved addon nodes under the sidebar, a consistent wrap-aware addon scroll-range helper, and tests covering the scroll decision.
</objective>
