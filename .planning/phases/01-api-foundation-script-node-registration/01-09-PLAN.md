<objective>
Close two ADDON-node rendering gaps, both isolated to `NodeGraph.java`:
- GAP-4 (minor): the addon node body preview (script text drawn by the addon's `AddonNodeBodyRenderer`) draws on top of the slideover sidebar category panel. Root cause: `renderAddonNodeContent` (NodeGraph.java:7392) calls `renderer.render(ctx, context, x+1, y+18, width-2, height-19)` with no scissor, so the addon's raw `drawTextWithShadow` text can paint outside the node and over the panel that animates over the workspace. The sidebar's open/expanded rendered width is already known to NodeGraph via `sidebarWidthForRendering` (set each frame from `sidebar.getWidth()` in PathmindVisualEditorScreen).
- GAP-5 (cosmetic): with the addon jar absent, orphaned ADDON nodes render as a blank grayed body (`renderAddonPlaceholderBody`, NodeGraph.java:7424) with no indication of which addon is missing or that the node is a no-op. The node still retains `addonTypeId` (D-09), so the placeholder can name it.

Purpose: the addon node preview behaves like a well-clipped first-class node, and the graceful-degradation story (API-09) is legible to users instead of looking like a rendering bug.
Output: scissor-clipped, sidebar-overlap-aware addon body rendering, and a missing-addon indicator label in the placeholder body.
</objective>
