<objective>
Close NEW-CR-01 (Warning, recommended before Phase 2): `AddonLoader.failedAddons` is a `static final LinkedHashMap` written during single-threaded mod init but read from the UI thread via `getFailedAddons`/`getFailure` (D-08 failure UX). `markFailed` is `public static`, so addon code can call it from arbitrary threads. A `LinkedHashMap` is not thread-safe — concurrent read/write can corrupt internal structure and raise `ConcurrentModificationException` when the UI thread iterates the unmodifiable view while another thread writes.

Purpose: Make the failure-tracking collection thread-safe without breaking the D-08 failure-surface UI, which may rely on insertion order. Phase 2 increases concurrency (worker-thread addon execution), so this hardening lands before that surface area grows.
Output: `failedAddons` becomes `Collections.synchronizedMap(new LinkedHashMap<>())`, preserving insertion order; no public API or caller changes.
</objective>
