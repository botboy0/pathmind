<action>
Add a SEPARATE vertical scroll for the category icon bar, mirroring the content-panel scroll fields (74-76). Steps:
1. Add fields: `private int iconBarScrollOffset = 0;` and `private int iconBarMaxScroll = 0;` (and a drag flag only if you implement scrollbar-thumb dragging; wheel + click-track is sufficient).
2. In `render` tab-size logic (661-677): stop shrinking tabs toward 8px to force-fit. Keep `tabSize = TAB_SIZE` (allow a modest floor like 20 only if width-constrained), compute `totalTabHeight = totalVisibleTabs*tabSize + (totalVisibleTabs-1)*tabSpacing`, set `iconBarMaxScroll = max(0, totalTabHeight - availableTabHeight)`, and clamp `iconBarScrollOffset` to `[0, iconBarMaxScroll]`.
3. Offset every tab Y by `- iconBarScrollOffset`: built-in `tabY` (765) and addon `addonTabY` (807). Wrap the icon-bar tab drawing in a scissor clip to `[sidebarStartY+TOP_PADDING, sidebarStartY+TOP_PADDING+availableTabHeight]` so scrolled tabs clip cleanly instead of bleeding into the content panel.
4. When `iconBarMaxScroll > 0`, draw a thin scrollbar in the icon-bar column (reuse the content-panel scrollbar style/colors) with thumb size/position derived from `availableTabHeight`, `totalTabHeight`, `iconBarScrollOffset`.
5. Apply the SAME `- iconBarScrollOffset` offset to the tab click hit-tests (769-770 and 811-812) so a scrolled-into-view icon selects the correct category.
6. In `mouseScrolled` (~1218): when the cursor X is within the icon-bar column (x ≤ currentInnerSidebarWidth / the icon strip width) AND `iconBarMaxScroll > 0`, adjust `iconBarScrollOffset` by the wheel delta (clamp), and return handled — do NOT also scroll the content panel in that case.
</action>
