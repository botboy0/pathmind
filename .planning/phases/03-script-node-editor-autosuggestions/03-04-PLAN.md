Two co-located vertical slices on the editor body: the line-number gutter (EDIT-02) and the persisted last-run error strip (EDIT-03). The gutter is a draw pass left of the EditBoxWidget that stays synchronized with the editor's content and scroll. The error strip is a bottom-of-body band that shows `⚠ Line N: <message>` (full message on hover) when `lastError` is set, and is absent when the last run succeeded — with `lastError`/`lastErrorLine` persisted in the schema-versioned extra-fields blob (bumped to v2) and originated from the Cobalt `LuaError` at execution time.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Cobalt LuaError → error strip/tooltip | Lua-VM error message (from untrusted script) rendered into the node body strip + hover tooltip |
| extra-fields blob (disk) → renderer | Persisted lastError/lastErrorLine loaded from a preset file and displayed |
| worker thread → game thread | Error write-back crosses the worker→game-thread boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04-01 | Tampering / Injection | Error message rendered as a format string or markup | mitigate | Error drawn via `drawTextWithShadow` as literal text (no `String.format`, no Minecraft Text formatting codes interpreted from the raw message). Truncation is plain substring. No injection surface. |
| T-03-04-02 | Tampering | Malformed lastErrorLine in a loaded preset (negative/huge/non-number) | mitigate | Deserialize via `instanceof Number ? intValue() : 0`; renderer treats the value as a label only (no array indexing by it). Safe default 0. |
| T-03-04-03 | Denial of Service | Pathologically long error message bloats the tooltip/strip | mitigate | Strip truncates to width-8 + "..."; tooltip word-wraps to 220px max (UI-SPEC). No unbounded single-line draw. |
| T-03-04-04 | Tampering | Worker-thread write to node state races the render thread | mitigate | Write-back marshaled to the game thread via `MinecraftClient.getInstance().execute(...)` (RESEARCH open-question #1) — no cross-thread node mutation. |
| T-03-04-05 | Tampering | Schema v2 blob loaded by an OLDER addon build | accept | Schema version is read tolerantly; unknown keys are ignored by older code; v1↔v2 is additive (script field unchanged). Client-side; low risk. |
| T-03-04-SC | Tampering | npm/pip/cargo installs | accept | No package installs; no new dependencies (RESEARCH Package Legitimacy Audit: none). |
</threat_model>
