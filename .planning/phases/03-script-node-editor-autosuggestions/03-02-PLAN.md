<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Minecraft screen → addon input handler | Key/char/mouse/scroll events forwarded from the trusted MC screen to a third-party addon's `AddonNodeInputHandler` |
| NodeGraph graph shortcuts ↔ focused editor | Destructive graph keys (Delete/Backspace node, Esc close, arrow pan) must not fire while the editor is focused |
| extraFields blob ↔ node identity | A `_node_id` UUID and `lastError` are written into the schema-versioned opaque blob loaded from disk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Denial of Service / Tampering | Key leak: typing in editor deletes/pans the graph | mitigate | `handleAddonInputKeyPressed` inserted BEFORE Esc-close + Delete/Backspace gates; handler returns true for ALL keys while focused (RESEARCH Pitfall 2). UAT-verified in 03-05. |
| T-03-02-02 | Tampering | Hostile addon abuses input hook (e.g. swallows ALL input even unfocused) | mitigate | NodeGraph only routes to the handler when `focusedAddonNode != null`; an unfocused/unfocusable addon never receives key/char/scroll. Click routing is gated to ADDON-type node body hits only. |
| T-03-02-03 | Tampering / DoS | A buggy/hostile `renderOverlay` or input handler throws and crashes the editor frame | mitigate | `renderOverlay` wrapped in the same try/catch-log-and-skip used for `render` (NodeGraph 7422-7426). Input forwarders should tolerate handler exceptions (do not propagate). |
| T-03-02-04 | Tampering | Malformed `_node_id`/`lastError`/`lastErrorLine` in a loaded preset blob | mitigate | Read defensively via instanceof/Number.intValue() with safe defaults (null/0); regenerate `_node_id` if absent or non-String. No eval, no reflection on blob contents. |
| T-03-02-05 | Information Disclosure | nodeId leaks sensitive data | accept | nodeId is a random UUID with no semantic content; client-side only; no PII. |
| T-03-02-SC | Tampering | npm/pip/cargo installs | accept | No package installs; no new external dependencies (RESEARCH Package Legitimacy Audit: none). |
</threat_model>
