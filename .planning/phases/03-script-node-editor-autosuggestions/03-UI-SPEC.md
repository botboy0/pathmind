# Phase 3 — UI Design Contract: Script Node Editor + Autosuggestions

> Visual and interaction contract for the in-game Lua script editor rendered
> inside the Pathmind node body via `DrawContext` (immediate-mode). This is
> NOT a web or HTML UI. No CSS, no component library. Every measurement is
> in pixels at GUI scale 2 (Minecraft default). All color values are UITheme
> constants expressed as packed ARGB integers.
>
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — in-game `DrawContext` immediate-mode rendering |
| Preset | not applicable |
| Component library | Minecraft `EditBoxWidget` (multiline, 1.21.4) + Pathmind `UITheme` tokens |
| Icon library | Unicode glyphs only (⚠) — no icon atlas |
| Font | Minecraft default font via `TextRenderer`; `textRenderer.fontHeight` = 9 px; no monospace available |

shadcn gate: not applicable. This is a Java/Fabric Minecraft mod, not a web
or React project. There is no web tooling in scope.

## Spacing Scale

All values are in pixels at GUI scale. Must be multiples of 4 except where
a Minecraft built-in constant forces otherwise (noted).

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 2 px | Gutter divider width; caret horizontal margin inside line |
| sm | 4 px | Editor internal text padding (left of gutter, left of code text); popup row left margin |
| md | 8 px | Gutter-to-text divider column gap; error strip vertical padding; popup panel padding |
| lg | 10 px | Line height per editor row (fontHeight 9 + 1 px leading) — matches existing `LINE_HEIGHT = 10` in `LuaScriptNodeRenderer` |
| xl | 16 px | Node body header height (inherited from NodeGraph: header occupies `y+1` to `y+18`) |
| 2xl | 18 px | Error strip height (`fontHeight` 9 + 2×md/2 padding); matches `UITheme.INPUT_HEIGHT = 18` |

Exceptions:
- Editor body height is fixed at **108 px** (8–10 visible lines × 10 px line height + 4 px top/bottom internal padding = 108 px fits 10 lines). This matches `TEMPLATE_NODE_HEIGHT = 108` already in `Node.java` and gives the Script node a well-known fixed height without inventing new constants.
- Gutter width is dynamic: 1-digit lines → 14 px; 2-digit → 20 px; 3-digit → 26 px. Formula: `digitCount × 6 px (char width) + 8 px (4 px left + 4 px right padding)`. Uses `Node.CHAR_PIXEL_WIDTH` equivalents; recalculate each frame from line count.
- Node total height = header (18 px) + editor body (108 px) + error strip (18 px, when present) + 2 px bottom border = **146 px** (error visible) or **128 px** (error absent). The addon must call the node dimension API (or the addon API surface) to set this at registration time; the exact mechanism is Claude's discretion during planning.
- Popup suggestion rows: 14 px each (matches `NavigatorChatSuggestions.ENTRY_HEIGHT = 14`).

## Typography

Minecraft's `TextRenderer` is the only available text primitive. There are no
font-weight or font-size controls; the font is fixed at 9 px ascent height
(`textRenderer.fontHeight`). Typography is expressed as functional roles.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Editor code text | 9 px (`fontHeight`) | normal (only weight available) | 10 px (fontHeight + 1) | Every line of Lua code in the editor body |
| Gutter number | 9 px (`fontHeight`) | normal | 10 px | Line numbers in the gutter; rendered in `UITheme.TEXT_TERTIARY` (dimmed) |
| Error strip text | 9 px (`fontHeight`) | normal | 9 px (single line, no leading needed) | `⚠ Line N: <message>` in error strip |
| Suggestion label | 9 px (`fontHeight`) | normal | 14 px (ENTRY_HEIGHT) | Function name in the popup |
| Suggestion signature | 9 px (`fontHeight`) | normal | 14 px (ENTRY_HEIGHT) | Signature hint (`(x, y, z)`) rendered to the right of the label |

Shadow rendering: use `drawTextWithShadow` for all text to match the rest of
the Pathmind UI. Do NOT use `drawText` (no shadow).

## Color

All tokens are `UITheme` constants from
`com.pathmind.ui.theme.UITheme`. Hex values are recorded here for reference;
the implementation MUST reference the constant name, not the hex literal (per
the UITheme Javadoc: "ALL UI colors must be defined here — no hardcoded hex
values in rendering code").

### Editor Body

| Surface | UITheme Token | Hex | Usage |
|---------|---------------|-----|-------|
| Editor background (unfocused) | `BACKGROUND_INPUT` | `0xFF101010` | Editor text area fill when not focused |
| Editor background (focused) | `NODE_INPUT_BG_ACTIVE` | `0xFF181818` | Editor text area fill when focused — matches existing in-node field active state |
| Editor border (unfocused) | `BORDER_SUBTLE` | `0xFF2E2E2E` | 1 px border around editor area, unfocused |
| Editor border (focused) | `BORDER_FOCUS` | `0xFF575757` | 1 px border around editor area, focused |
| Editor code text | `TEXT_PRIMARY` | `0xFFE7E7E0` | All Lua code lines |
| Editor placeholder text | `TEXT_TERTIARY` | `0xFF6A6A6A` | Placeholder shown when script is empty and editor is unfocused |
| Caret | `CARET_COLOR` | `0xFFE6F7FF` | Text insertion caret; blinking at 530 ms interval (matches existing caret blink in NodeGraph) |
| Text selection highlight | `TEXT_SELECTION_BG` | `0x805577D6` | Selection rectangle behind selected text |

### Gutter

| Surface | UITheme Token | Hex | Usage |
|---------|---------------|-----|-------|
| Gutter background | `BACKGROUND_SECONDARY` | `0xFF1A1A1A` | Fills the gutter column |
| Gutter line numbers | `TEXT_TERTIARY` | `0xFF6A6A6A` | Dimmed right-aligned line number text |
| Gutter divider | `BORDER_SUBTLE` | `0xFF2E2E2E` | 1 px vertical line between gutter and code text |

### Error Strip

| Surface | UITheme Token | Hex | Usage |
|---------|---------------|-----|-------|
| Error strip background | `BACKGROUND_SECONDARY` | `0xFF1A1A1A` | Strip fill below editor body |
| Error strip border (top) | `BORDER_DANGER_MUTED` | `0xFF6A4A4A` | 1 px top separator line between editor and error strip |
| Error text | `STATE_ERROR` | `0xFFC62828` | `⚠ Line N: <message>` in error red, matching `NodeErrorNotificationOverlay` palette |
| Error strip absent | _(no surface rendered)_ | — | When `lastError` is null, the strip area is simply not drawn; node height contracts by 18 px |

### Autosuggestion Popup

| Surface | UITheme Token | Hex | Usage |
|---------|---------------|-----|-------|
| Popup background | `PANEL_BACKGROUND` (local: `0x80000000`) | `0x80000000` | Semi-transparent panel fill; matches `NavigatorChatSuggestions.PANEL_BACKGROUND` |
| Popup border | `BORDER_HIGHLIGHT` | `0xFF676767` | Panel outline; matches `NavigatorChatSuggestions` border call |
| Default row background | _(none — transparent)_ | — | Non-selected rows have no fill |
| Selected/hovered row | `0x503A3A3A` | `0x503A3A3A` | Selected row highlight; matches `NavigatorChatSuggestions` selected row fill exactly |
| Suggestion function name | `TEXT_HEADER` | `0xFFF5F4EE` | Primary label text; matches `NavigatorChatSuggestions` |
| Suggestion signature hint | `TEXT_SECONDARY` | `0xFFA0A39C` | Signature/hint text to the right of the function name; matches `NavigatorChatSuggestions` |

### 60/30/10 Distribution (adapted for in-game context)

| Role | Dominant 60% | Secondary 30% | Accent 10% |
|------|-------------|----------------|------------|
| Surfaces | `BACKGROUND_INPUT` / `NODE_INPUT_BG_ACTIVE` (editor body) | `BACKGROUND_SECONDARY` (gutter + error strip + popup) | `CARET_COLOR` (caret), `TEXT_SELECTION_BG` (selection), `BORDER_FOCUS` (focused border), `STATE_ERROR` (error text) |

Accent reserved for:
1. Focused editor border (`BORDER_FOCUS`) — indicates keyboard capture
2. Text selection highlight (`TEXT_SELECTION_BG`) — active selection only
3. Caret (`CARET_COLOR`) — insertion point only
4. Error strip text and icon (`STATE_ERROR`) — error state only
5. Selected suggestion row (`0x503A3A3A`) — active popup selection only

Destructive color (`STATE_ERROR` `0xFFC62828`): error strip text ONLY. Not used for any other element in this phase.

## Visual States

Every interactive surface must be specified for all states the executor will encounter.

### Editor Body States

| State | Border | Background | Description |
|-------|--------|------------|-------------|
| Unfocused, empty script | `BORDER_SUBTLE` | `BACKGROUND_INPUT` | Shows placeholder text `-- click to edit` in `TEXT_TERTIARY` |
| Unfocused, has content | `BORDER_SUBTLE` | `BACKGROUND_INPUT` | Shows first N visible lines of script text in `TEXT_PRIMARY`; gutter shows line numbers |
| Focused | `BORDER_FOCUS` | `NODE_INPUT_BG_ACTIVE` | Caret visible, blinking; all keystrokes captured; gutter still visible |
| Focused, has selection | `BORDER_FOCUS` | `NODE_INPUT_BG_ACTIVE` | Selection highlight (`TEXT_SELECTION_BG`) behind selected region; caret at selection end |
| Over sidebar (dimmed) | `BORDER_SUBTLE` (grayscale) | `NODE_INPUT_BG_DIMMED` | Matches NodeGraph pattern for over-sidebar dimming; editor is non-interactive in this position |

### Error Strip States

| State | Visible | Content |
|-------|---------|---------|
| No error (last run succeeded or never run) | No | Strip not rendered; node height is 128 px |
| Error present (`lastError` non-null) | Yes | `⚠ Line N: <message>` truncated to strip width − 8 px; full message on hover; node height is 146 px |

### Autosuggestion Popup States

| State | Visible | Description |
|-------|---------|-------------|
| Closed | No | Not rendered |
| Open, no selection | Yes | First item highlighted by default (`selectedIndex = 0`) |
| Open, item selected | Yes | Selected row has `0x503A3A3A` fill; `selectedIndex` tracks arrow key or mouse hover |
| Open, mouse hover | Yes | Hovering a row updates `selectedIndex` to that row (same highlight color) |

## Layout

### Node Body Layout (top to bottom, all values relative to node origin)

```
y + 0                   ┌─────────────────────────────────────┐
                        │  Node header (18 px, rendered by    │
y + 18                  │  NodeGraph — not this phase)         │
                        ├──────┬──────────────────────────────┤
                        │Gutter│  Editor text area             │
                        │14-26 │  (width - gutter - 2 border) │
                        │  px  │                               │
                        │      │  EditBoxWidget or custom      │
                        │      │  draw pass                    │
                        │      │                               │
y + 18 + 108 = y + 126  ├──────┴──────────────────────────────┤
                        │  Error strip (18 px)                 │
                        │  ⚠ Line N: <message>                 │
y + 144                 └─────────────────────────────────────┘
                        +2 px bottom border = 146 px total
```

When error strip is absent, node height = 128 px (18 header + 108 editor + 2 border).

### Autosuggestion Popup Layout

```
┌─────────────────────────────────────┐
│  getBlock        (x, y, z)          │  ← selected row (0x503A3A3A fill)
│  getInventory    ()                  │
│  getPosition     ()                  │
│  getVar          (name)              │
│  moveTo          (x, y, z)           │
│  setVar          (name, value)       │
└─────────────────────────────────────┘
        ↑ anchored below cursor token
```

Popup dimensions:
- Width: 220 px (matches `NavigatorChatSuggestions.WIDTH = 220`)
- Entry height: 14 px (matches `NavigatorChatSuggestions.ENTRY_HEIGHT = 14`)
- Padding: 4 px top + 4 px bottom (matches `NavigatorChatSuggestions.PADDING = 4`)
- Max rows shown: 6 (cap per CONTEXT.md decision)
- Position: anchored at pixel-x of cursor token start, pixel-y just below current line; clamped to node body scissor bounds

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "click to edit" (placeholder text when script is empty and editor unfocused) |
| Empty state heading | _(none — placeholder text is the empty state)_ |
| Empty state body | `-- click to edit` rendered in `TEXT_TERTIARY` at the editor's first line position |
| Error state | `⚠ Line N: <message>` — N is the 1-based Lua line number from the error; message is the Lua error string, truncated with `...` if it overflows the strip width minus 8 px |
| Error hover (tooltip) | Full error message, word-wrapped to 220 px max, via `TooltipRenderer.render(...)` |
| Destructive confirmation | None in this phase — the Script node has no destructive user actions |

### Copywriting Rules

- Error format string: `"⚠ Line " + lineNumber + ": " + message`
- Truncation: strip available width − 8 px; append `"..."` if truncated
- Placeholder: exactly `-- click to edit` (a valid Lua comment, doubles as a hint)
- Suggestion signature format: `functionName` in `TEXT_HEADER`, then `(param1, param2, ...)` in `TEXT_SECONDARY` separated by 12 px gap, matching `NavigatorChatSuggestions` layout. For zero-param functions: `()`.
- Symbol display names (auto-derived from `PathmindBindings.build(...)`):
  - `getVar(name)`
  - `setVar(name, value)`
  - `moveTo(x, y, z)`
  - `getPosition()`
  - `getInventory()`
  - `getBlock(x, y, z)`
  - Lua keywords (static list): `and`, `break`, `do`, `else`, `elseif`, `end`, `false`, `for`, `function`, `goto`, `if`, `in`, `local`, `nil`, `not`, `or`, `repeat`, `return`, `then`, `true`, `until`, `while`
  - Lua stdlib (static list, minimal for v1): `print`, `tostring`, `tonumber`, `type`, `pairs`, `ipairs`, `next`, `select`, `unpack`, `table.insert`, `table.remove`, `table.concat`, `string.format`, `string.len`, `string.sub`, `math.floor`, `math.ceil`, `math.abs`, `math.max`, `math.min`

## Interaction Contract

### Focus Model

| Action | Effect | Key consumed? |
|--------|--------|---------------|
| Click inside editor area | Editor gains focus; `BORDER_FOCUS` border; keystroke routing active | N/A (mouse) |
| Esc while focused (no popup) | Editor loses focus; `BORDER_SUBTLE` border; keystrokes route to NodeGraph again | Yes — consume Esc before NodeGraph |
| Click outside editor area | Editor loses focus (same as Esc) | N/A (mouse) |
| Any keypress while focused | Route to `EditBoxWidget`; do NOT pass to NodeGraph | Yes — all keys consumed while focused |

Leak-proofing is the primary UAT risk (CONTEXT.md). The input-routing hook
in the new addon API must return `true` from all `keyPressed` / `charTyped`
handlers while the editor is focused, preventing NodeGraph from processing
those events (node nudge, delete, pan).

### Autosuggestion Interaction

| Action | Effect | Key consumed? |
|--------|--------|---------------|
| Type `pathmind.` | Open popup; show all `pathmind.*` symbols | No — dot is inserted normally |
| Ctrl+Space | Open popup; show symbols matching prefix left of cursor | Yes |
| Arrow Up / Down (popup open) | Move `selectedIndex`; clamp to [0, size−1] | Yes |
| Mouse hover over row | Set `selectedIndex` to hovered row index | N/A (mouse) |
| Mouse click on row | Accept suggestion at clicked row; close popup | N/A (mouse) |
| Enter (popup open) | Accept currently selected suggestion; insert completion; close popup | Yes — do NOT insert newline |
| Esc (popup open) | Close popup without accepting | Yes — do NOT blur editor |
| Any other key (popup open) | Route to `EditBoxWidget` as normal edit; update prefix; refilter popup | No |
| Type until prefix has no matches | Close popup | No |

Acceptance behavior: replace the incomplete token left of the cursor with the
selected function name only (not the signature). E.g., if user typed
`pathmind.getB` and accepts `getBlock`, the text becomes `pathmind.getBlock`
with cursor placed after the closing character.

### Scrolling

- Vertical scroll inside editor: `EditBoxWidget` handles via `mouseScrolled`
  forwarded by the new input-routing hook. No horizontal scrollbar drawn
  (no soft-wrap; text clips at node edge under scissor).
- Horizontal scroll: `EditBoxWidget` handles internally; no visible
  scrollbar indicator in v1. Long lines simply extend beyond the visible
  viewport and scroll with left/right arrow or End key.

## Focus-Leak Affordances

These are visual indicators that the editor holds keyboard focus and
keystrokes will not reach the graph:

1. **Border change**: unfocused `BORDER_SUBTLE` → focused `BORDER_FOCUS`
   (2×2 step up in brightness). This is the primary affordance.
2. **Background change**: `BACKGROUND_INPUT` → `NODE_INPUT_BG_ACTIVE`
   (matches the existing in-node field editing pattern in NodeGraph ~line 3761).
3. **Caret visible**: blinking caret at cursor position confirms edit mode.

No additional focus-indicator overlay is needed; the border + background +
caret combination is consistent with existing Pathmind in-node editing
patterns.
