## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user keyboard/clipboard → EditBoxWidget | Untrusted Lua text typed/pasted into the editor (clipboard paste via native EditBoxWidget) |
| Pathmind input forwarder → addon handler | Key/char/mouse/scroll forwarded only while this node holds focus |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-01 | Denial of Service / Tampering | Key leak: editing deletes/pans the node graph | mitigate | keyPressed/charTyped return true for ALL keys while focused; Esc-blur consumes (returns true). UAT-verified in 03-05 (SC#5). This is the primary UAT risk. |
| T-03-03-02 | Tampering | Malicious clipboard paste (huge text / control chars) | mitigate | `setMaxLength(Integer.MAX_VALUE)` allows large scripts but the timeout/sandbox (Phase 2 LUA-04) bounds execution; editor only stores text — it does not execute on type. Control chars handled by EditBoxWidget natively (no markup interpretation). |
| T-03-03-03 | Information Disclosure | Editor text rendered as a format string | accept | Lua text is drawn via `drawTextWithShadow` as literal text — no `String.format`/markup; no injection surface. |
| T-03-03-04 | Tampering | Stale widget position causing clicks to land wrong / escape node bounds | mitigate | Set widget x/y/width BEFORE every render and before mouseClicked (Pitfall 1/3); NodeGraph scissor clips overflow. |
| T-03-03-SC | Tampering | npm/pip/cargo installs | accept | No package installs; EditBoxWidget is bundled MC; Cobalt already present from Phase 2 (RESEARCH Package Legitimacy Audit: none). |
