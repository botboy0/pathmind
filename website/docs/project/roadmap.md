### Open gates before calling v1 done

- [x] **Phase 3 in-game UAT (SC#5)** — the 6-point editor checklist, **6/6 automated & passing** (2026-07-10). Points 1–5 via `specs/lua-editor-uat.yaml` (run `20260710-095656`); point 6 — error strip on failure, cleared on success — via the new `specs/lua-error-strip.yaml` (run `20260710-090058`, 35 steps: place node, replace default script with one failing on line 2, wire start→node by port drag, run with `K`, assert chat `Lua error: script:2:` + red `⚠ Line 2` strip, fix script, rerun, assert strip cleared). The spec exposed **three real cross-repo blockers**, all fixed same-day: (1) editor input handlers got a throwaway context and nothing wrote script edits back to the node — typed scripts were silently dropped and execution ran the serializer default; (2) execution runs on branch clones, so executor error write-backs never reached the workspace node — errors now route via a `_node_id`-keyed `AddonRuntimeErrors` bridge; (3) `EditBoxWidget`'s constructor `message` param is narration, not content — loaded scripts rendered as an empty editor (plus the serializer round-trip dropped Pathmind-managed `_node_id`). Two testkit robustness fixes fell out too (drag-coordinate parsing, held keypresses for tick-polled keybinds).
- [x] **Phase 2 UAT formally confirmed** (2026-07-10) — full `pathmind.*` surface exercised in-game via the automated harness (`testing/testruns/20260710-001707/`, RESULT: PASS, 7/7 presets): variable round-trips for number/string/boolean + absent→nil + table rejection (`uat-lua-vars`), `getPosition`/`getBlock` loaded + unloaded→nil/`getInventory` shape (`uat-lua-gamestate`), awaitable `moveTo` with exact arrival 12 blocks out (`uat-lua-moveto`), uncaught error stops the graph with `script:3:` line number in chat (`uat-lua-error-expectfail`), and runaway loop killed by the 5s compute budget with `script:2:` in chat (`uat-lua-timeout-expectfail`).

## Infrastructure

- [x] mc-testkit: prebuilt-jar loadout replaces the in-container Pathmind build (2026-07-10) — a source-less Loom "devshell" project baked into the image provides `runClient` (fabric-api + architectury as dev-mapped runtime libs); **both** jars (pathmind-fabric + addon) arrive via the loadout overlay and are runtime-remapped by Fabric Loader (mixins + access widener included). Spec runs no longer depend on the WSL clone or a per-run mod build; the entrypoint falls back to mod-source mode when a repo is mounted at `/workspace` (`WORKSPACE_SRC=…`). - ✅ **Automated in-game test flow via HeadlessMC** (2026-07-09, retired 2026-07-10): built all jars, launched MC 1.21.4 headless via a harness mod, executed fixture presets, wrote `results.json` per run — used to close the Phase 2 UAT gate. Removed on 2026-07-10 together with the play/drive scripts; GUI testing now runs exclusively through the containerized vision testkit below.
- ✅ **Containerized vision-driven GUI test pipeline** (2026-07-10): `testing/` (own repo, [botboy0/mc-testkit](https://github.com/botboy0/mc-testkit)) runs the dev client GPU-rendered in Docker on WSL2 (Mesa d3d12 → host RTX via `/dev/dxg`, hard renderer assertion, no llvmpipe) and executes natural-language YAML test steps via a vision model over OpenRouter (screenshot → grounded click/key via xdotool). Per run: `run.mp4` + `results.json` with per-step cost/tokens/timing. E2E proven: inventory + Pathmind editor opened and visually asserted, ~$0.002/run, warm boot ~20 s. This is the sole GUI test flow; the mod repo keeps only standard unit tests. Docs: `testing/README.md` + the in-game testing guide.
- ✅ **Node-failure observability + expected-fail presets** (2026-07-10): node failures complete their futures normally and only surfaced as UI notifications, making them invisible to automation — `ExecutionManager.recordNodeFailure` (fed by `NodeExecutionCompletion.fail`) now exposes a queryable failure count/message, the harness fails a preset when one is recorded, and presets named `*-expectfail` invert pass/fail for negative-path fixtures. This immediately exposed the silently-broken `print` call in `smoke-lua` and the `moveTo` timeout bug.

- ✅ **Testkit: loadouts, commentary mode, run dashboard** (2026-07-10): mc-testkit gained per-run persistent artifacts + a web dashboard (run list, live status, per-step video seeking, `https://` LAN access); **loadouts** (mount a mods/config overlay tree incl. `.wipe` state reset — the `pathmind-lua` jar now loads in the containerized dev client via Fabric runtime remapping, closing the "no addon in runClient" gap); and **commentary mode** (model reports visual oddities independent of verdicts). Commentary immediately surfaced the two editor rendering bugs now in the v2 backlog. Docs: `testing/README.md` + the [in-game testing guide](../guides/in-game-testing.md).

- [x] **mc-testkit: vision backends — assert-model split + subscription broker + dashboard card** (2026-07-11, branch `vision-backends` in the testkit repo): the driver now separates *judgment* from *grounding* — `ASSERT_MODEL` (spec: `assert_model`) runs `assert` verdicts and commentary on a stronger model while the cheap grounder keeps doing `act` clicks; failing asserts take their second opinion from the escalation model directly. New host-side `tools/vision-broker.py` (stdlib, OpenAI-compatible) routes by model prefix: `openrouter/…` (default, unchanged), `claude/…` via Claude Pro/Max subscription OAuth (`claude setup-token`; requires the `oauth-2025-04-20` beta header and the Claude Code identity line as first system block), `codex/…` via the Codex CLI's ChatGPT OAuth (Responses API over SSE, rotating refresh tokens persisted back; only codex-tier models, e.g. `gpt-5.6-sol`). Both subscription paths verified end-to-end **with image input** — vision asserts on subscription plans instead of API credit (cost 0). `BROKER=1 run-local.sh …` wires containers through the broker and defaults the three model roles to `auto-*` aliases from `~/.mc-testkit/broker.json`; the run dashboard gained a broker card (per-backend health + alias dropdowns via a `/api/broker` proxy), so the next run's models are switchable from the browser. 14 new unit tests (driver split, message conversion, routing, token refresh, webui proxy).

### Lua runtime
- [x] Generic `invokeAction(name, args)` dispatch (2026-07-10) — `pathmind.invokeAction` invokes any world/player/interface action node by case-insensitive type name with parameter args (scalars, case-insensitive names, unknown names fail loudly with the valid list); flow control/sensors/data/parameter nodes stay graph-only. Pathmind side: `PathmindRuntime.invokeAction` + `AddonActionInvoker` (synthetic node through the normal `NodeCommandDispatcher` path on the client thread). Documented in the new [Lua Scripting Reference](../guides/lua-scripting.md).
- [x] Table variable marshaling (2026-07-11) — `getVar`/`setVar` now cross Lua tables: `{x,y,z}` tables ↔ coordinate variables (`PARAM_COORDINATE`), array tables ↔ **runtime lists** (the Create List / Add to List / List Item namespace; elements uniformly numbers/strings/booleans/coordinate tables; coordinate entries read back as `{x,y,z}` tables, opaque entries like entity UUIDs as strings). `PARAM_DISTANCE` variables also read as numbers now. API contract stays plain-Java (`Map`/`List` on `PathmindRuntime`); shape errors (empty/sparse/mixed-key tables, mixed lists) fail loudly as Lua errors. New: `AddonListEntryCodec` (shared `pm_list:` wire format), `ExecutionManager.get/setRuntimeListForAnyActiveChain`, `LuaMarshal` addon-side. Unit-tested both sides (`PathmindRuntimeImplTableMarshalingTest`, `PathmindBindingsTableTest`); in-game spec `lua-table-marshal.yaml`. Userdata stays unsupported (the sandbox can't construct any). Documented in the [Lua Scripting Reference](../guides/lua-scripting.md).
- [ ] Per-node timeout override.
- [ ] Real instruction-budget sandboxing beyond the compute-time timeout.
- [x] Fix the off-thread `whenComplete` touching Minecraft client state (2026-07-10) — addon executor futures complete on the addon's worker thread, so every downstream chain continuation ran off the client thread (all other node types complete inside `client.execute`). `Node.executeAddonNode` now hops to the client thread before completing the chain future.
- [x] Navigator: short `moveTo` hops rejected (2026-07-10) — `searchPlanningTarget` discarded every non-exact path shorter than 8 entries, including the near-goal partials `findPathToGoal` deliberately returns; for short hops every legitimate path is short. The same ≤6-block proximity exception now applies before the floor. Verified in-game by the new `lua-moveto-short-hop.yaml` spec (4-block hop on a flat platform, run `20260710-190540`). Note: steep/obstructed terrain can still fail short hops later in the pipeline (`isViablePlannedPath` — "invalid movement sequence"), which is legitimate no-route behavior.
- [x] Re-arm the timeout timer after awaitable actions (2026-07-10) — `CobaltVm` now runs a periodic compute-aware watchdog with a `blocked` flag raised around action futures; also fixes the safety-net `Thread.interrupt()` killing legitimate `moveTo` waits longer than ~5.5s wall-clock.
- [x] Provide a `print` binding in the Lua sandbox (2026-07-10) — sandbox-safe global `print` in `CobaltVm`/`PathmindBindings.buildPrint`, tab-joins arguments via Cobalt's `toStringDirect` and routes the line to player chat; the serializer default script now runs cleanly on a fresh node (unit-tested in `CobaltVmSmokeTest`).

### Editor

_The items closed on 2026-07-10 (print binding, strip first-line, strip dimming, focus re-gain, popup anchor) are verified in-game by the mc-testkit specs `lua-v2-print-dim.yaml` (run `20260710-180204`) and `lua-editor-uat.yaml` (run `20260710-183026`), alongside the unit tests in `CobaltVmSmokeTest` / `LuaScriptNodeRendererTextTest` / `PopupAnchorTest`._

- [x] Cursor-aware suggestion acceptance (2026-07-10) — the addon's first mixin (`EditBoxWidgetAccessor`, new `pathmind-lua.mixins.json`) exposes `EditBoxWidget`'s private `EditBox` model, giving the editor the real cursor offset: `getLineBeforeCursor` now reads the cursor's actual line, acceptance replaces the token at the cursor and restores the cursor after the inserted name (`computeAcceptance`, unit-tested in `SuggestionAcceptanceTest`), the popup anchors to the cursor's line instead of the last line, and the CR-04 end-of-text safe-fail guard is gone — mid-script completion works. Verified in-game by the new `lua-cursor-accept.yaml` spec (run `20260710-202547`): completion on the middle line of a three-line script edits that line in place, the popup anchors below it, and follow-up typing lands right after the inserted name (`pathmind.getBlock(1)`).
- [x] Evict stale entries from the per-node editor-state map (2026-07-10, closes WR-04) — new addon-API lifecycle hook `AddonNodeInputHandler.onNodeRemoved(nodeId)` (default no-op): NodeGraph fires it from `removeNodeInternal` (all discrete deletion paths) and before the bulk clears in preset load/import, undo/redo restore, and workspace clear; a focused node is blurred first so `focusedAddonNode` can't dangle. The Lua addon evicts its `EditorState` there. Documented in the addon getting-started guide ("Node lifecycle").
- [x] Syntax highlighting (2026-07-11) — addon-side: `LuaHighlighter` (single-pass tokenizer emitting absolute-offset spans; keywords/strings/comments/numbers/`pathmind.*`+`print` builtins; long brackets `[=[…]=]` and long comments span lines; unterminated literals highlight to EOL/EOF) + `HighlightingEditBoxWidget`, which overrides `renderContents` — the designated `protected abstract` override point on `ScrollableTextFieldWidget`; vanilla draws plain strings at hardcoded `0xFFE0E0E0` with no styling hook — and redraws per-line colored segments plus its own caret/selection (span cache keyed on text). Needed the addon's first access widener (`EditBox$Substring` is a protected nested record). Palette keyed to the muted dark theme (amber keywords, green strings, tertiary-grey comments, light-blue numbers, sky-blue builtins). Unit-tested (`LuaHighlighterTest`); in-game: `lua-editor-v2.yaml` run `20260710-233731`, `syntax-highlighting.png`.
- [x] Script hot-reload without graph restart (2026-07-11) — Pathmind-side only, zero addon changes: execution runs on branch clones frozen at chain start, so edits never reached a running chain. New `AddonLiveScripts` registry (forward analogue of `AddonRuntimeErrors`, keyed by the stable `_node_id`): `NodeGraph.buildAddonContext` republishes the workspace script every rendered frame/input event and `syncAddonContextToNode` right after each edit; `Node.executeAddonNode` prefers the live text over the clone's frozen `script` field. Granularity is per-execution — an in-flight script is never interrupted; the next execution (loop iteration) picks up the edit. **Pitfall found by the spec:** the first cut evicted entries in `onNodeRemoved`, which also fires on graph-replacement paths — closing the editor mid-run wiped the entry and execution fell back to the frozen snapshot; eviction now happens only on discrete deletion, and the per-frame republish self-heals the channel after preset load/undo/redo. Unit-tested (`AddonLiveScriptsTest`); in-game: `lua-editor-v2.yaml` run `20260710-233731` (`hot-reload-works.png` — script edited behind a running 25 s Wait prints the NEW text, OLD never appears). Documented in the [Lua Scripting Reference](../guides/lua-scripting.md).
- [x] Editor panel gap below the node header (2026-07-11, user report) — the addon body rect started at the `HEADER_HEIGHT` layout offset (y+18) while the drawn header bar is only 14 px + 1 px edge, leaving a 3 px strip of node background between header and editor panel; `NodeGraph` now hands addon renderers y+15 / height−16 (render, overlay, scissor, placeholder), so the panel sits flush under the header.
- [x] Script text bled over nodes dragged across the editor (2026-07-11, user report) — two stacked z-order mechanics: DrawContext text goes into fixed buffers that only flush at end-of-frame (so glyphs paint over later nodes' fills), and *shadowed* text additionally renders its foreground glyphs at z+0.03 (winning the depth test against later same-z fills even when flushed early). Fix: `NodeGraph` flushes the batch right after every addon body render (inside the scissor), and the editor draws all in-body text shadowless at z=0 (`drawText` — also matches Pathmind's own `drawNodeText` style); popup/tooltip overlays keep their shadow and stay on top by design. In-game regression spec: `lua-editor-zorder.yaml` run `20260710-234655` (`zorder-covered.png`).
- [ ] Built-in node label text can still bleed over overlapping nodes (same deferred-text mechanics, pre-existing; visible when node headers overlap). Needs a per-node batch flush or per-node z-stratification in `renderNode`.
- [x] Handle `\n` in error tooltips (2026-07-10) — `wrapLines` replaces the flatten-to-spaces wrap: newlines now start new tooltip lines, traceback tabs become a two-space indent, and the cap (raised 6 → 8 lines) appends an `…` line when content is cut (unit-tested in `WrapLinesTest`).
- [x] Suggestion popup anchored one row too high (2026-07-10) — the anchor was a fixed offset from the editor top; it now sits one row below the line being typed using the gutter's pixel model (last line × 9 px, scroll-corrected; `computePopupAnchorY`, unit-tested in `PopupAnchorTest`). Verified in-game: `lua-editor-uat` run `20260710-183026`, `suggestions-dot.png` shows all three script lines clear of the popup.
- [x] ~~Editor `=` glyph rendering artifact~~ resolved as **not a bug** (2026-07-10) — the "crossed box" is the X11 mouse pointer captured in the container screenshots, parked wherever the vision model last clicked (usually mid-line, next to `=`). Cross-run evidence: the identical glyph appears between `local` and `x` (no `=` nearby) in run `20260710-175701` step 34, and run `20260710-183026` `suggestions-dot.png` shows crisp `=` glyphs with the pointer visible beside them. Vision-commentary reports of this shape can be disregarded.
- [x] Error strip multi-line message → box glyph (2026-07-10) — the strip now renders only the first line of the error (`LuaScriptNodeRenderer.firstLine`, unit-tested); the full traceback stays available in the hover tooltip.
- [x] Editor focus lost after Esc-blur (2026-07-10, found by the new `lua-v2-print-dim` vision spec) — after blurring the editor with Esc, clicking back into it never re-focused: NodeGraph's WR-01 guard skips `focusAddonNode` when it still considers the node focused, and the addon's Esc self-blur is invisible to NodeGraph, so `onFocusGained` never re-fired and keystrokes leaked to the graph (Esc closed the whole screen). The addon now takes focus directly in its `mouseClicked` handler (idempotent with `onFocusGained`).
- [x] Error strip stale after editing (2026-07-10) — the strip now dims (subtle border + tertiary text) once the script differs from the snapshot taken when the error was recorded (`EditorState.isErrorStale`); it stays red while the script matches the failing run. Known limit: a re-run of an edited script producing a byte-identical error message keeps the strip dimmed (errors arrive value-only via the context, so a new run with the same message is indistinguishable).
